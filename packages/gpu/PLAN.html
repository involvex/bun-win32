⚡ The performance doctrine (this is a GPU engine — read before every hot-path phase)

A GPU engine that leaves cycles on the table is a contradiction. Performance is a standing, always-on audit, not a closing benchmark — every hot-path phase ends by analyzing its inner loop down to allocations, FFI-boundary crossings, the COM vtable call, and CPU cycles. Treat every number below as a floor, not a finish line — while building, keep actively hunting for wins this plan never anticipated; the listed levers are where to start, not where to stop (see the generative pass, G.3). The rules (measured on the authoring machine 2026-06-09 — Bun runs on JavaScriptCore, not V8; re-verify per host, and treat V8 folklore as suspect until measured; full data in Appendix G):

  • Reuse, never re-allocate, in any per-dispatch / per-readback / per-frame loop. A Buffer.alloc per iteration measured ~1064 ns; reuse is ~free (~1600×); a fresh DataView per parse is +20 ns. Allocate desc/scratch buffers and views ONCE (the proven core-scope.ts / _gpu.ts pattern — the D3D11_BOX and desc buffers are module/loop-hoisted). run() and readbackBuffer must not allocate a staging resource or a new Float32Array(bytes) copy per call where a reused buffer or a view suffices — pool them.
  • Cache buffer.ptr once. The .ptr getter re-runs ptr(this) on every access (~7.3 ns) — ~19% off a real FFI loop. Never write nativeFn(buf.ptr) inside GpuArray.from/readbackBuffer/the dispatch path; hoist the pointer.
  • Mind the vcall hot path — it is the hottest code in the engine. The invoker builds a template-string key `${method}|${returns}|${argTypes.join(',')}` and does a Map.get on EVERY COM call — per-call string allocation in the inner loop. Microbench it (S5.x); if it shows up, key the cache more cheaply (e.g. by the resolved method pointer) — but only with a measurement, and never change vcall's argument contract (the this-excluded rule is load-bearing, ground truth).
  • Avoid bigint in the hot path; minimize FFI crossings. The FFI floor is ~5 ns/call; a u64→bigint return adds ~10 ns; a fresh BigInt() arg per call is ~16 ns (hoist constant handle args out of loops). Pass Number(x) as Pointer (the sanctioned narrowing) rather than re-boxing. The whole compute model is the macro-opt: one dispatch over a big buffer, not N tiny calls.
  • Use DataView typed accessors, not hand-rolled byte math (on JSC manual byte-OR is ~1.13× slower — the opposite of V8 lore); Big(U)int64Array views are the slowest u64 path and often unaligned-illegal; JSC shows no monomorphism penalty — don't contort for it. Spend the effort on alloc + .ptr + the readback loop.
  • The readback cliff is real. Readback synchronizes the GPU (stated in the mental model). The per-byte read.u8 loop in readbackBuffer is not just to A/B once (S4.2) — it must be eliminated (bulk copy / view) and the staging buffer reused; and the async-readback extra (Phase 10b #2) is the structural fix for the stall. Pace windowed demos with present(true) vsync, never Bun.sleep (async sleep is ~15.6 ms quantized; Bun.sleepSync is NOT, ~1.5 ms — a useful distinction the plan should not blur).
  • run() has a compile-per-call cliff. The one-shot API compiles HLSL + allocates + releases every call (S8.1 finally). Flag it loudly, wire the shader bytecode cache (Phase 10b #9) into run(), and ship a Kernel/GpuArray-reuse recipe so the hot path compiles once and pools resources.
  • Every hot path gets an isolated microbenchmark (Bun.nanoseconds(), 200k+ warm-up, dead-code sink, median ns/op — the terminal.bench.ts convention) — single dispatch, single readback element, vcall overhead, run() per-element, cbuffer write. Benchmarks DRIVE optimization, not just the README table. A perf baseline is recorded in Phase 0 and gated at release (S14.x) exactly like a tsc error.
  • Prove it, don't hope — Bun ships bun:jsc. Because this is Bun-only, every hot path can be verified optimized: numberOfDFGCompiles(fn) > 0 (timing optimized code, not the interpreter), profile(cb) tier breakdown (a warmed loop measured 99.3% FTL on this machine — assert FTL/DFG dominates the dispatch/readback decode), reoptimizationRetryCount(fn) (climbing = type-unstable deopt — fix the polymorphism, e.g. the vcall call sites, not the body), jscDescribe(obj) (Structure/IndexingType X-ray), and heapStats().objectTypeCounts deltas (gate per-dispatch/per-readback allocation at zero). Full verified toolkit + measured shape/array/alloc/native-read levers in Appendix G.4/G.5 — use them in every microbench and audit.
  • Don't satisfice — beating the baseline is the job, not meeting it. Correctness plus the known levers is the START of a hot-path audit, never the end. Once a path is correct, profile() it to find the ACTUAL dominant cost (often something this plan never listed — the vcall cache-key build, a redundant readback copy, a per-call compile), form a hypothesis, microbench an alternative, and keep it only if it wins AND stays correct, cast-free, and readable — then go again until returns genuinely diminish. Interrogate every read, branch, allocation, dispatch, and pass: can it be hoisted, precomputed, batched, fused, made async/lazy, or eliminated? Can the algorithm change (a better tiling, fewer barriers, one dispatch instead of many), not just the constant factor? The S14.6 gate only enforces "no regression"; the standing goal is to beat the prior baseline every phase. Any lever you discover that this plan didn't list — prove it with the G.4 tools (and the GPU timestamp queries for kernel-ms) and add it to Appendix G.5. Never stop at "fast enough."

const invokers = new Map<string, ReturnType<typeof CFunction>>();

/** Invoke COM method `slot` on `thisPtr`; argTypes/args exclude the implicit `this`. */
export function vcall(thisPtr: bigint, slot: number, argTypes: readonly FFIType[], args: readonly unknown[], returns: FFIType = FFIType.i32): number {
  const vtable = read.u64(Number(thisPtr) as Pointer, 0);
  const method = read.u64(Number(vtable) as Pointer, slot * 8);
  const key = `${method}|${returns}|${argTypes.join(',')}`;
  let invoke = invokers.get(key);
  if (invoke === undefined) {
    invoke = CFunction({ ptr: Number(method) as Pointer, args: [FFIType.u64, ...argTypes], returns });
    invokers.set(key, invoke);
  }
  return invoke(thisPtr, ...args) as number;
}

const INCLUDE_PATTERN = /^[ \t]*#include[ \t]+"([^"]+)"[ \t]*$/gm;

/** Resolve #include directives textually (no ID3DInclude callback — foreign-thread COM hazard, repo rule). */
export function preprocessHLSL(source: string, includes: Record<string, string> = {}, depth = 0): string {
  if (depth > 8) throw new Error('preprocessHLSL: #include nesting exceeds 8 levels (cycle?).');
  return source.replace(INCLUDE_PATTERN, (line, name: string) => {
    const body = includes[name];
    if (body === undefined) throw new Error(`preprocessHLSL: unresolved #include "${name}". Pass it via CompileOptions.includes — file-system includes are not resolved.`);
    return preprocessHLSL(body, includes, depth + 1);
  });
}

Phase 8 — kernel.ts: run() / Kernel / GpuArray — the flagship API

Purpose: the gpu.js-successor surface. This phase is where competitive parity is won (see Appendix D): one-shot ergonomics, retained GPU-resident arrays for kernel chaining without readback (gpu.js "pipeline" mode), multiple outputs (gpu.js "kernel maps"), uniforms, 1D/2D/3D dispatch, automatic WARP fallback — typed end to end with zero casts.

API design commitments (frozen here, validated against the Phase 0 competitive research):
  • run(source, buffers, options?) — one-shot: compile → upload → dispatch → read back → release. Mutates the caller's typed arrays in place for writable bindings and returns the same record. In-place is the cast-free, type-exact design (run<T>(…, buffers: T): T holds trivially) and means zero aliasing surprises.
  • GpuArray — a retained GPU buffer (UAV + SRV) that persists across dispatches. The chaining primitive.
  • Kernel — a compiled shader bound to parsed bindings; dispatch() accepts only GpuArrays (explicit lifecycle); run() is the sugar that wraps typed arrays for you.
  • Bindings are parsed textually from explicit register(uN/tN/bN) annotations — deterministic, unit-testable without a GPU, and the error messages can quote the exact line to add. (COM reflection via D3DReflect stays available for diagnostics, but the hot path never depends on it.) v1 constraint: registers must be dense from 0 per kind; enforced with a clear error.
  • Element types: float/int/uint ↔ Float32Array/Int32Array/Uint32Array, stride 4. Struct-typed buffers remain a low-level makeStructuredBuffer affair in v1 (documented recipe).

const BUFFER_PATTERN = /\b(RW)?StructuredBuffer\s*<\s*(float|int|uint)\s*>\s+([A-Za-z_]\w*)\s*:\s*register\(\s*([tu])(\d+)\s*\)/g;
const CBUFFER_PATTERN = /\bcbuffer\s+\w+\s*:\s*register\(\s*b0\s*\)/;
const NUMTHREADS_PATTERN = /\[\s*numthreads\s*\(\s*(\d+)\s*,\s*(\d+)\s*,\s*(\d+)\s*\)\s*\]/;

/** Parse explicit register() buffer declarations. Throws actionable errors — this is the API contract, not a heuristic. */
export function parseKernelBindings(source: string): KernelBinding[] {
  const bindings: KernelBinding[] = [];
  for (const match of source.matchAll(BUFFER_PATTERN)) {
    const [, readWrite, kind, name, space, register] = match;
    const writable = readWrite === 'RW';
    if (writable && space !== 'u') throw new Error(`Kernel buffer "${name}": RWStructuredBuffer must use a u register — write : register(u${register}).`);
    if (!writable && space !== 't') throw new Error(`Kernel buffer "${name}": StructuredBuffer must use a t register — write : register(t${register}).`);
    bindings.push({ kind: kind === 'float' ? 'float' : kind === 'int' ? 'int' : 'uint', name, register: Number(register), writable });
  }
  if (bindings.length === 0) throw new Error('Kernel source declares no StructuredBuffer/RWStructuredBuffer with an explicit register() annotation. Example: RWStructuredBuffer<float> data : register(u0);');
  for (const writable of [true, false]) {
    const registers = bindings.filter((binding) => binding.writable === writable).map((binding) => binding.register).sort((a, b) => a - b);
    registers.forEach((register, index) => {
      if (register !== index) throw new Error(`Kernel ${writable ? 'u' : 't'} registers must be dense from 0 (found ${writable ? 'u' : 't'}${register} where ${writable ? 'u' : 't'}${index} was expected).`);
    });
  }
  return bindings;
}

export function parseNumthreads(source: string): readonly [number, number, number] {
  const match = NUMTHREADS_PATTERN.exec(source);
  if (match === null) throw new Error('Kernel source has no [numthreads(x,y,z)] attribute on its entry point.');
  return [Number(match[1]), Number(match[2]), Number(match[3])];
}

/** A GPU-resident array (UAV + SRV). Survives across dispatches — the kernel-chaining primitive. */
export class GpuArray {
  #resource: StructuredBuffer;
  readonly kind: ScalarKind;
  readonly length: number;

  private constructor(resource: StructuredBuffer, kind: ScalarKind, length: number) {
    this.#resource = resource;
    this.kind = kind;
    this.length = length;
  }

  static alloc(kind: ScalarKind, length: number): GpuArray {
    ensureDevice();
    return new GpuArray(makeStructuredBuffer({ count: length, srv: true, stride: 4, uav: true }), kind, length);
  }

  static from(data: KernelArray): GpuArray {
    ensureDevice();
    const initialData = Buffer.from(data.buffer, data.byteOffset, data.byteLength);
    return new GpuArray(makeStructuredBuffer({ count: data.length, initialData, srv: true, stride: 4, uav: true }), kindOf(data), data.length);
  }

  get srv(): bigint { return this.#resource.srv!; }
  get uav(): bigint { return this.#resource.uav!; }

  /** Read back into a caller-owned typed array (cast-free typed reads). */
  readInto<A extends KernelArray>(target: A): A {
    const bytes = readbackBuffer(this.#resource.buffer, this.length * 4);
    if (kindOf(target) !== this.kind) throw new Error(`GpuArray.readInto: target is ${kindOf(target)} but the array holds ${this.kind}.`);
    target.set(this.kind === 'float' ? new Float32Array(bytes) : this.kind === 'int' ? new Int32Array(bytes) : new Uint32Array(bytes));
    return target;
  }

  read(): KernelArray {
    const bytes = readbackBuffer(this.#resource.buffer, this.length * 4);
    return this.kind === 'float' ? new Float32Array(bytes) : this.kind === 'int' ? new Int32Array(bytes) : new Uint32Array(bytes);
  }

  release(): void {
    comRelease(this.#resource.uav ?? 0n);
    comRelease(this.#resource.srv ?? 0n);
    comRelease(this.#resource.buffer);
  }
}

export interface DispatchOptions {
  /** Thread groups [x, y, z]. Default: [ceil(maxBoundLength / numthreadsX), 1, 1]. */
  groups?: readonly [number, number?, number?];
  /** Raw bytes for the cbuffer at register(b0) (use cbufferLayout() to build them). */
  uniforms?: Buffer | KernelArray;
}

export class Kernel {
  #bindings: readonly KernelBinding[];
  #constantBuffer = 0n;
  #shader: bigint;
  #threads: readonly [number, number, number];
  #usesUniforms: boolean;

  constructor(source: string, options: CompileOptions = {}) {
    ensureDevice();
    this.#bindings = parseKernelBindings(source);
    this.#threads = parseNumthreads(source);
    this.#usesUniforms = CBUFFER_PATTERN.test(source);
    this.#shader = makeComputeShader(compile(source, 'main', 'cs_5_0', options));
  }

  dispatch(buffers: Record<string, GpuArray>, options: DispatchOptions = {}): void {
    const uav: bigint[] = [];
    const srv: bigint[] = [];
    let maxLength = 1;
    for (const binding of this.#bindings) {
      const array = buffers[binding.name];
      if (array === undefined) throw new Error(`Kernel.dispatch: no GpuArray passed for buffer "${binding.name}".`);
      if (array.kind !== binding.kind) throw new Error(`Kernel.dispatch: buffer "${binding.name}" is declared ${binding.kind} but the GpuArray holds ${array.kind}.`);
      (binding.writable ? uav : srv)[binding.register] = binding.writable ? array.uav : array.srv;
      if (array.length > maxLength) maxLength = array.length;
    }
    const cb: bigint[] = [];
    if (options.uniforms !== undefined) {
      if (!this.#usesUniforms) throw new Error('Kernel.dispatch: uniforms were passed but the kernel declares no cbuffer at register(b0).');
      const bytes = Buffer.isBuffer(options.uniforms) ? options.uniforms : Buffer.from(options.uniforms.buffer, options.uniforms.byteOffset, options.uniforms.byteLength);
      if (this.#constantBuffer === 0n) this.#constantBuffer = makeConstantBuffer(bytes.byteLength);
      updateConstantBuffer(this.#constantBuffer, bytes);
      cb.push(this.#constantBuffer);
    }
    const [x = Math.ceil(maxLength / this.#threads[0]), y = 1, z = 1] = options.groups ?? [];
    csSet(this.#shader, { cb, srv, uav });
    dispatch(x, y ?? 1, z ?? 1);
  }

  release(): void {
    comRelease(this.#constantBuffer);
    comRelease(this.#shader);
  }
}

export interface RunOptions extends DispatchOptions {
  compile?: CompileOptions;
}

/** One-shot: compile, upload, dispatch, read back (in place), release. The 10-line-wow API. */
export function run<T extends Record<string, KernelArray>>(source: string, buffers: T, options: RunOptions = {}): T {
  const kernel = new Kernel(source, options.compile);
  const uploaded = new Map<string, GpuArray>();
  try {
    const bound: Record<string, GpuArray> = {};
    for (const binding of kernel.bindings) {
      const data = buffers[binding.name];
      if (data === undefined) throw new Error(`run: kernel declares buffer "${binding.name}" but no typed array was passed for it.`);
      const array = GpuArray.from(data);
      uploaded.set(binding.name, array);
      bound[binding.name] = array;
    }
    kernel.dispatch(bound, options);
    for (const binding of kernel.bindings) {
      if (binding.writable) uploaded.get(binding.name)!.readInto(buffers[binding.name]!);
    }
    return buffers;
  } finally {
    for (const array of uploaded.values()) array.release();
    kernel.release();
  }
}

export type CBufferFieldType = 'float' | 'float2' | 'float3' | 'float4' | 'float4x4' | 'int' | 'int2' | 'int3' | 'int4' | 'uint' | 'uint2' | 'uint3' | 'uint4';

type CBufferValue<F extends CBufferFieldType> = F extends 'float' | 'int' | 'uint' ? number : readonly number[];

const FIELD_BYTES: Record<CBufferFieldType, number> = { float: 4, float2: 8, float3: 12, float4: 16, float4x4: 64, int: 4, int2: 8, int3: 12, int4: 16, uint: 4, uint2: 8, uint3: 12, uint4: 16 };

export interface CBufferLayout<T extends Record<string, CBufferFieldType>> {
  byteSize: number;
  offsets: { readonly [K in keyof T]: number };
  write(values: { readonly [K in keyof T]: CBufferValue<T[K]> }): Buffer;
}

/** HLSL packing: fields align to 4 bytes, never straddle a 16-byte register; float4x4 aligns to 16; total rounds up to 16.
 *  Matrices: HLSL defaults to column-major — pass transposed data or declare row_major in the shader. */
export function cbufferLayout<T extends Record<string, CBufferFieldType>>(fields: T): CBufferLayout<T> {
  const offsets: Record<string, number> = {};
  let cursor = 0;
  for (const [name, type] of Object.entries(fields)) {
    const bytes = FIELD_BYTES[type];
    if (type === 'float4x4') cursor = Math.ceil(cursor / 16) * 16;
    else if (Math.floor(cursor / 16) !== Math.floor((cursor + bytes - 1) / 16)) cursor = Math.ceil(cursor / 16) * 16;
    offsets[name] = cursor;
    cursor += bytes;
  }
  const byteSize = Math.ceil(cursor / 16) * 16;
  return {
    byteSize,
    offsets,
    write(values) {
      const out = Buffer.alloc(byteSize);
      for (const [name, type] of Object.entries(fields)) {
        const offset = offsets[name]!;
        const value = values[name];
        const integer = type.startsWith('int') ? 'int' : type.startsWith('uint') ? 'uint' : 'float';
        if (typeof value === 'number') {
          if (integer === 'int') out.writeInt32LE(value, offset);
          else if (integer === 'uint') out.writeUInt32LE(value, offset);
          else out.writeFloatLE(value, offset);
        } else {
          value.forEach((component, index) => {
            if (integer === 'int') out.writeInt32LE(component, offset + index * 4);
            else if (integer === 'uint') out.writeUInt32LE(component, offset + index * 4);
            else out.writeFloatLE(component, offset + index * 4);
          });
        }
      }
      return out;
    },
  };
}

G.3 — The discipline

  1. Baseline (S0.6b) records the FFI/dispatch/readback floors on the release host before any work.
  2. Every hot-path phase (4, 5, 8) ends with a microbenchmark step recording ns/op and asserting zero per-dispatch/per-readback allocation.
  3. Release gate (S14.6) re-runs everything; a >~10% regression vs baseline OR any forbidden hot-loop pattern blocks release.
  4. README numbers (GFLOPS/MB-s/µs) come only from the gated run — never copied from this appendix.
  5. Generative pass (the real spirit — do this every hot-path phase). An audit does not just apply G.5's levers and check the gate. It profile()s the path (plus GPU timestamp queries for kernel-ms) to locate the true hot spot, then actively tries at least one optimization this plan did NOT anticipate — a fused dispatch, a reused staging buffer, a cheaper vcall key, a precomputed layout, a barrier removed, a better tiling — measures it, and folds any proven win back into G.5. "Meets baseline" is the floor; the work is to beat it. Stop only at real diminishing returns, never at "good enough." The agent that ships the fastest correct engine, not the one that merely passes the gate, did the job.
  6. Methodology (verified on Bun 1.4.0/JSC, this machine): Bun.nanoseconds(); warm ≥200k iterations (1k is too few) to trigger tier-up; assert numberOfDFGCompiles(fn) > 0 and abort if 0 (else you're timing LLInt/Baseline); folded sink to defeat dead-code elimination; vary offsets per-iteration to defeat hoisting; gcAndSweep() between (not within) trials; median of ≥5 — the terminal.bench.ts convention. For GPU-side timing, the GPU timestamp queries (Phase 10b #1) give honest kernel-ms; wall-clock CPU timing alone over-counts the sync.
